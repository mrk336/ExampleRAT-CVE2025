ExampleRAT-CVE2025

A modular Remote Access Tool (RAT) written in C# for red team simulation and malware analysis.

Disclaimer

This project is intended strictly for research and educational use. It simulates malware behavior to help security professionals understand threat mechanics. Do not deploy this code in unauthorized systems.

Why I Built This

I wanted to explore how threat actors design modular malware that can persist, evade detection, and execute payloads dynamically. This project helped me understand key concepts in Windows internals, encryption, and sandbox evasion. It also gave me a chance to practice writing clean, secure code for offensive tooling.

How It Works

Key Derivation

Uses PBKDF2 with SHA-512 and 10,000 iterations to derive a 256-bit AES key from a passphrase.

Limitation: No salt is used, which weakens cryptographic strength. A random salt should be added in production.

Payload Decryption

Each payload is stored as a Base64 string and decrypted using AES-CBC. The first 16 bytes are used as the IV.

Limitation: The code slices 15 bytes for the IV, which is incorrect. AES requires a full 16-byte IV.

Persistence Module

Decrypted payload is written to a temp file and executed via WMIExec.

Limitation: Assumes WMIExec.exe is present. No fallback or validation.

Sandbox Evasion

Introduces a randomized delay (2–8 seconds) to mimic human-like behavior and evade sandbox detection.

Limitation: Basic technique. Could be improved with virtualization checks or user interaction simulation.

Exfiltration Module

Same structure: decrypt, write, execute.

Limitation: No actual exfiltration logic implemented. Placeholder only.

Sandbox Detection Module

Final payload execution simulates sandbox detection.

Limitation: No actual detection logic included.

Sample Payloads

You can replace the embedded Base64 strings with your own encrypted binaries. Example using OpenSSL:

openssl enc -aes-256-cbc -in payload.exe -out encrypted.bin -K <key> -iv <iv>

Then convert the encrypted binary to Base64 and embed it in the code.

How to Run

git clone https://github.com/mrk336/ExampleRAT-CVE2025
cd ExampleRAT-CVE2025
dotnet build

Note: WMIExec.exe must be available in your system path.
